# FORGE Obsidian Vault Generator

Generate a fully-featured Obsidian vault from FORGE documents without modifying the original FORGE structure.

## Overview

The vault generator transforms FORGE's agent-optimized markdown files into an Obsidian vault with:

- **YAML frontmatter** - Metadata, tags, and phase tracking
- **Wiki-links** - Convert file paths to `[[Architecture]]` style links
- **MOCs** - Maps of Content for navigation (Active Changes, Phase Dashboard, Architecture Index)
- **Dataview queries** - Dynamic task status and phase filtering
- **Graph styling** - CSS snippets for phase-based node colors
- **Canvas files** - Visual system overview
- **Templates** - For manual notes

## Quick Start

```bash
# Generate vault at default location (.forge-vault/)
just vault

# Generate at custom location
just vault ~/Documents/ObsidianVaults/my-project

# Force full regeneration (useful after script updates)
just vault --force

# Preview changes without writing files
python forge/scripts/generate_vault.py --output .forge-vault --dry-run
```

## Vault Structure

```
.forge-vault/
  .obsidian/              ← Obsidian config (workspace, graph, plugins, CSS)
  _meta/                  ← MOCs and dashboards (GENERATED)
    00-README.md          ← Vault entry point
    01-Active-Changes.md  ← Active changes overview
    02-Architecture-Index.md
    03-Phase-Dashboard.md ← Dataview-powered dashboard
  _custom/                ← Your manual notes (PRESERVED)
  _templates/             ← Obsidian templates
  _canvas/                ← Visual maps
  core/                   ← FORGE docs (GENERATED)
    FORGE-Protocol.md
    CLAUDE-Manual.md
    Architecture.md
    Constraints.md
    Verification.md
  changes/                ← All changes (GENERATED)
    {change-id}/
      _index.md           ← Change overview
      spec.md
      tasks.md
      decisions.md
```

## Smart Merge Behavior

The generator uses smart merge to preserve your manual additions:

### GENERATED Files (Always Overwritten)

Files marked with `<!-- GENERATED by forge/scripts/generate_vault.py -->`:
- All files in `core/`
- All files in `changes/*/`
- All files in `_meta/`
- `.obsidian/` config files
- `_canvas/` files

### PRESERVED Files (Never Touched)

- `_custom/` - Your manual notes
- `_templates/` - Only created if missing, never overwritten

## Features

### 1. YAML Frontmatter

Every generated file includes metadata:

```yaml
---
type: specification
change_id: "add-notifications"
phase: "execute"
tags: [forge/spec, phase/execute, change/add-notifications]
aliases: ["add-notifications spec"]
source: forge/changes/add-notifications/spec.md
generated: 2026-03-30T02:02:00.592491+00:00
total_requirements: 8
---
```

### 2. Wiki-Links

File references are converted to wiki-links:

```markdown
# Before (FORGE source)
Read `forge/global/architecture.md` before writing anything.

# After (vault)
Read [[Architecture]] before writing anything.
```

### 3. Graph View

Nodes are color-coded by phase:
- **Blue** - Plan phase
- **Orange** - Review phase
- **Green** - Execute phase
- **Red** - Blocked
- **Gray** - Done

### 4. Dataview Queries

The Phase Dashboard includes dynamic queries:

```dataview
TABLE 
  total_tasks as "Tasks",
  completed as "Done",
  pending as "Pending",
  blocked as "Blocked"
FROM "changes"
WHERE type = "task-list" AND phase = "execute"
SORT file.name ASC
```

### 5. MOCs (Maps of Content)

Navigate through curated entry points:
- **00-README.md** - Vault overview
- **01-Active-Changes.md** - Current work status
- **02-Architecture-Index.md** - Core docs
- **03-Phase-Dashboard.md** - Visual status board

## Workflow

### First Time Setup

1. Generate the vault:
   ```bash
   just vault
   ```

2. Open `.forge-vault/` in Obsidian

3. Enable the Dataview plugin (Community Plugins → Dataview)

4. Explore the vault starting from `_meta/00-README.md`

### Regular Updates

After making changes to FORGE documents or completing tasks:

```bash
just vault
```

The vault will be updated with the latest state while preserving your custom notes in `_custom/`.

### Adding Custom Notes

Create notes in the `_custom/` directory:

```bash
# They will be preserved across vault updates
.forge-vault/_custom/my-notes.md
.forge-vault/_custom/meeting-2026-03-30.md
```

### Force Regeneration

If the vault gets corrupted or you want a clean slate:

```bash
just vault --force
```

**Warning:** This deletes the entire vault and recreates it. Custom files in `_custom/` will be lost.

## Obsidian Setup

### Required Plugins

- **Dataview** - For dynamic queries in the Phase Dashboard

### Recommended Settings

1. **Graph View**
   - Enable "Show tags"
   - Use color groups (automatically configured)

2. **File Explorer**
   - Sort by name
   - Show file extensions

3. **Editor**
   - Enable "Strict line breaks"
   - Enable "Auto pair brackets"

## Customization

### Adding Custom CSS

Add CSS snippets to `.obsidian/snippets/`:

```css
/* .obsidian/snippets/my-custom.css */
.graph-view.color-fill[data-tag*="custom"] { 
  fill: #ff00ff; 
}
```

Enable in Settings → Appearance → CSS snippets

### Modifying Templates

Edit templates in `_templates/`:
- `manual-note.md` - Template for custom notes
- `meeting-note.md` - Template for meeting notes

These are only created if missing, so you can safely modify them.

## Troubleshooting

### Vault won't open in Obsidian

Make sure you're opening the `.forge-vault/` directory, not a file inside it.

### Dataview queries not working

1. Install the Dataview plugin
2. Enable it in Settings → Community Plugins
3. Reload Obsidian

### Custom files disappeared

If you used `--force`, custom files are deleted. Always use the default `just vault` command for updates.

### Wiki-links not resolving

Make sure the target file exists. Check the source path in the frontmatter.

`forge/scripts/generate_vault.py`

```bash

# Generate to test location
python forge/scripts/generate_vault.py --output /tmp/test-vault

```

### Modifying the Generator

The script is organized into methods:
- `_transform_core_docs()` - Transform FORGE.md, CLAUDE.md, etc.
- `_transform_changes()` - Transform change directories
- `_generate_mocs()` - Generate Maps of Content
- `_generate_canvas()` - Generate canvas files
- `_convert_file_refs_to_wikilinks()` - Wiki-link conversion logic
